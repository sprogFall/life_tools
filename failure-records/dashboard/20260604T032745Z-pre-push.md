## 精简错误信息
- `npm --prefix dashboard run build` 在 Next.js 编译阶段失败。
- `src/app/layout.tsx` 使用 `next/font/google` 加载 `Fira Sans` 与 `Fira Code`，当前环境无法连接 Google Fonts 代理地址 `192.168.31.207:20171`，导致 `Failed to fetch` 并终止构建。

## 解决方案
- 根因：dashboard 构建期依赖远程 Google Fonts，网络不可用时会阻断静态导出。
- 修改 `dashboard/src/app/layout.tsx`，移除 `next/font/google` 导入和字体初始化。
- 修改 `dashboard/src/app/globals.css`，用本地系统字体栈声明 `--font-body`、`--font-heading`，并在 `<body>` 使用 `dashboard-fonts` 类保持字体变量入口。
- 修改 `dashboard/src/app/layout.test.tsx`，删除不再需要的 `next/font/google` mock。
- 验证命令：`npm --prefix dashboard test`、`npm --prefix dashboard run build`、`bash scripts/pre-push.sh` 均通过。

## 预防方案
- dashboard 不应在生产构建期依赖外部字体下载；新增字体时优先使用系统字体栈或随仓库部署的本地字体资源。
- 若确需远程字体，必须确认 CI/本地构建网络可访问，并补充构建失败记录中的可回退方案。
- 修改 `layout.tsx`、全局字体或构建脚本后，至少运行 `npm --prefix dashboard run build` 验证静态导出。
