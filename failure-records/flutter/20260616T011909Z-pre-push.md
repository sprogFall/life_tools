## 精简错误信息
- `flutter analyze` 在 `test/tools/work_photo/work_photo_camera_page_test.dart` 报 3 个 info：
  `camera` / `material` 多余导入，以及测试直接导入未在本包声明依赖的 `camera_platform_interface`。
- 影响范围仅为外拍助手相机预览测试文件的 import 与测试桩实现。

## 解决方案
- 根因：测试为了模拟 `CameraPlatform` 直接依赖 transitive package，触发
  `depend_on_referenced_packages`；同时部分类型可由其他导入提供，触发
  `unnecessary_import`。
- 修改 `WorkPhotoCameraService`，将“相机原始比例 + 设备方向 => 预览显示比例”的计算抽为
  `displayAspectRatioForOrientation`，测试直接覆盖该纯函数，不再导入
  `camera_platform_interface`。
- 修改 `test/tools/work_photo/work_photo_camera_page_test.dart`，删除多余导入，并将页面预览布局测试聚焦到
  `WorkPhotoCameraPreviewFrame`。
- 验证命令：`flutter test test/tools/work_photo/work_photo_camera_page_test.dart`。

## 预防方案
- Flutter 测试不要直接导入未在 `pubspec.yaml` 声明的传递依赖；优先通过生产代码中的纯函数或本包公开 API 测试逻辑。
- 新增测试桩后先跑 `dart format --output=none --set-exit-if-changed <改动文件>` 和
  `flutter analyze`，避免 import 类 info 阻断 `pre-push`。
