## 精简错误信息
- `Build Android APK` 的 Analyze、Test 两个分片均在 `flutter pub get` 阶段退出，后续 APK 构建未执行。
- Flutter 3.38.6 检测到 `receive_sharing_intent 1.8.1` 仅兼容 Swift Package Manager，但 CI 未启用对应 Flutter 配置。
- GitHub API 匿名配额已耗尽并返回 `403`，通过公开 Actions 页面确认真实构建结论与运行地址：
  `https://github.com/sprogFall/life_tools/actions/runs/30013718111`。

## 解决方案
- 在 `.github/workflows/build-apk.yml` 的 Analyze、Test、Build 三个任务中，在安装 Flutter 后执行
  `flutter config --enable-swift-package-manager`，再运行 `flutter pub get`。
- 在 `scripts/tests/build-apk_workflow_test.sh` 增加出现次数断言，保证三个任务不会遗漏该配置。
- 验证命令：
  - `bash scripts/tests/build-apk_workflow_test.sh`
  - `flutter config --enable-swift-package-manager`
  - `flutter pub get`
  - `bash scripts/pre-push.sh`

## 预防方案
- Flutter 或插件升级后，必须在 CI 使用的 Flutter 版本上执行一次不带缓存的 `flutter pub get`。
- 新增影响所有 Flutter 任务的初始化配置时，用 workflow 测试守护 Analyze、Test、Build 三处一致性。
- `post-push` 遇到 GitHub API 限流时，可只读访问公开 Actions 页面核对结论，不应将查询失败误判为构建失败或成功。
