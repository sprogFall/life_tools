## 精简错误信息
- 远端 `Run shard tests` 失败：`test/design/no_direct_icon_color_in_lib_test.dart` 检出 `lib/tools/work_photo/widgets/work_photo_item_bar.dart -> colors.foreground`，业务层仍直接使用 `Icon(color: ...)`。
- 远端 `Build APK` 失败：`android/app/src/main/kotlin/com/example/life_tools/MainActivity.kt:204:5 Missing return statement`，`saveImageToMediaStore` 的 `try` 块没有作为函数返回值返回。

## 解决方案
- 将外拍拍摄项状态图标由 `Icon(color: ...)` 改为 `IOS26ButtonIcon`，复用 `IOS26Button` 注入的前景色，避免业务层直接指定 `Icon` 颜色。
- 将 `MainActivity.saveImageToMediaStore` 中的写入逻辑改为 `return try { ... }`，确保成功路径返回保存后的相册路径，异常路径仍清理 MediaStore 记录并抛出给上层兜底。
- 验证命令：`env FLUTTER_BIN=/opt/flutter/bin/flutter /opt/flutter/bin/flutter test test/design/no_direct_icon_color_in_lib_test.dart`；`env FLUTTER_BIN=/opt/flutter/bin/flutter bash scripts/pre-push.sh --scope flutter --test-module work_photo`。

## 预防方案
- 外拍助手业务 UI 中需要使用 `IOS26Icon` / `IOS26ButtonIcon`，不要直接写 `Icon(color: ...)`。
- Android 原生代码新增 `String?` 返回函数时，成功分支需要显式 `return` 或使用表达式体，避免 Kotlin 块体函数只执行语句不返回。
